데이터베이스
DB(DataBase)
- 구조화된 정보, 데이터의 조직화된 모음, 일반적으로 컴퓨터에 전자적으로 저장이 되고 데이터베이스 관리 시스템에 의해 제어된다.

DBMS(DataBaseManagementSystem)
- 데이터베이스와 사용자 또는 프로그램 간의 매개체 역할을 하여 사용자가 정보의 구성, 검색, 수정, 삭제와 같은 관리를 할 수 있게 해줌
DBMS의 형태
1. 계층형 데이터베이스
2. 네트워크형 데이터베이스
3. 관계형 데이터베이스

DBMS 소프트웨어의 종류
- MySQL
- Microsoft SQL Server
- PostgreSQL
- Oracle

데이터베이스의 사용자 계정
- 데이터베이스의 생성 및 관리, 데이터의 삽입, 수정, 변경 및 삭제를 수행하는 데이터베이스 계정을 의미한다.

.관리자 계정(administrator)
- 데이터베이스의 생성과 관리를 담당하는 슈퍼 유저의 계정이며 정의, 생성, 삭제 등의 작업이 가능

일반 계정
- 데이터베이스에 대하여 명령(Query), 갱신, 보고서 작성 등의 작업을 수행할 수 있는 계정
- 일반 계정은 업무에 필요한 최소한의 권한만 가지는 것을 원칙으로 한다.

스키마
1. 데이터베이스의 구조와 제약조건에 관한 전반적인 명세를 기술한 메타 데이터의 집합이다.
2. 데이터베이스를 구성하는 데이터 개체(Entity),  속성(Attribute), 관계(Relationship) 및 데이터 조작시 데이터 값들이 갖는 제약 조건 등에 관해 전반적으로 정의한다.
3. 사용자의 관점에 따라 외부 스키마, 개념 스키마, 내부 스키마로 나누어진다.

스키마의 종류
1. 개념 스키마 : 전체적인 뷰
	ㄴ 조직체 전체를 관장하는 입장에서 DB를 정의한 것
	ㄴ 관계, 제약조건, 접근 권한, 보안 정책, 무결성 규칙에 관한 사항을 포함
2. 내부 스키마 : 물리적인 저장장치 입장에서 DB가 저장되는 방법을 기술한 것
	ㄴ 구체적인 개념으로 스키마를 디스크 기억장치에 물리적으로 구현하기 위한 방법을 기술한 것
	ㄴ 데이터베이스의 물리적 저장구조를 정의
	ㄴ 디스크에 어떤 구조로 저장할 것인가?
	ㄴ 데이터의 실제 저장 방법을 기술
	ㄴ 물리적인 저장장치와 밀접한 계층
3. 외부 스키마 : 사용자 뷰
	ㄴ 사용자나 개발자가 개인의 입장에서 필요한 데이터베이스의 논리적 구조를 정의
	ㄴ 실제로 존재하는 데이터들을 어떤 형식, 구조, 배치 화면을 통해 사용자에게 보여줄 것인가?
	ㄴ 하나의 데이터베이스에는 여러 개의 외부 스키마가 존재할 수 있다.

테이블
- 행과 열로 이루어진 데이터의 집합
- 일반적인 데이터베이스에서는 행과 열만 있으면 테이블이라고 하지만 관계형 데이터베이스에서는 특별한 제약을 추가해서 릴레이션(relation)이라고 부른다

행(row)
- 테이블을 구성하는 데이터들 중 가로로 묶은 데이터셋을 의미
- 일반적으로 한명에 대한 정보를 가지고 있다.
- 이 또한 관계형 데이터베이스에서는 튜플(tuple) 또는 레코드(rechord)라고 부른다
- 릴레이션에서 튜플의 개수를 카디널리티 라고 함.

열(column)
- 테이블을 구성하는 데이터들 중 세로로 묶은 데이터 셋을 의미
- 일반적으로 열은 그 테이블의 속성을 의미
- 열을 구성하는 값은 도메인(domain)으로 되어있다.
- 관계형 데이터베이스에서는 속성(attribute)라는 이름으로 부른다.
- 릴레이션에 열의 개수를 차수(degree)라고 한다.

도메인(domain)
- 하나의 속성이 취할 수 있는 동일한 유형의 데이터 집합
	ㄴ ex) 성별 -> 남, 여

SQL문
- 원하는 결과를 얻어오기 위해 DB에 요청하는 문장
- SQL 문은 대소문자를 구별하지 않는다.
- 한 줄 또는 여러 줄에 걸쳐 입력하는 것이 가능
- SQL 문장의 끝은 세미콜론으로 맺어야 한다.

SQL 문장의 종류
1. DDL(Data Definition Language) : 데이터 정의어
2. DML(Data Mainpulation Language) : 데이터 조작어
3. DCL(Data Control Language) : 데이터 제어어
4. TCL(Transaction Control Language) : 트랜잭션 제어어

DDL(Data Definition Language) : 데이터 정의어
	ㄴ 테이블 조작, 제어 관련 커리문
	ㄴ CREATE : 생성
	ㄴ DROP : 삭제
	ㄴ ALTER : 테이블 수정
		ㄴ 테이블명 수정 : ALTER TABLE 테이블명 RENAME TO  테이블명
		ㄴ 컬럼 추가 : ALTER TABLE 테이블명 ADD(컬럼명 자료형)
		ㄴ 컬럼명 변경 : ALTER TABLE 테이블명 RENAME COLUMN 컬럼명 TO 새로운 컬럼
		ㄴ 컬럼 삭제 : ALTER TABLE 테이블명 DROP COLUMN 컬럼명
	ㄴ TRUNCATE : 테이블에 있는 모든 데이터 삭제

제약조건
- 테이블에 문제가 되는 데이터가 입력되지 않도록 미리 지정해둔 조건
- 테이블을 생성할 때 함께 설정할 수 있고, 추후에 생성하거나 변경할 수도 있다.
- 되도록 테이블을 생성할 때 설정하는 것이 좋다.
- 데이터가 이미 들어간 상태에서는 변경하는 것이 매우 어렵다.

PRIMARY KEY	NULL값 불가능, 중복데이터 불가능, 테이블당 1개의 컬럼에 지정
FOREIGN KEY	NULL값 가능, 중복데이터 가능, 다른 테이블의 컬럼을 참조하여 해당 테이블에 존재하는 값만 입력 가능
UNIAUE		NULL값 가능, 중복되는 값 불가능
NOT NULL 	NULL값 불가능, 중복되는 값 가능
CHECK		- 설정한 조건을 만족하는 데이터만 입력 가능
		- 조건식을 만족하지 않는 데이터는 입력이 거부됨
DEFAULT		- 아무것도 입력하지 않으면 설정한 기본값이 입력됨
		- 값이 입력되면 기본값은 무시됨

NULL
- 정의되지 않는 값
- 빈 값 대신에 미정의 값을 부여할 때 사용 가능(PK는 불가능, FK는 가능)
